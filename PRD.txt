# Veteran Mentorship Platform (MVT) - Product Requirements Document

## Overview

MVT is a veteran mentorship platform designed as a replacement for the discontinued Veterati app. The platform connects military veterans for peer-to-peer mentorship, enabling experienced veterans to guide those seeking career advice, life coaching, and professional development.

## Technical Stack

- **Frontend**: Next.js 15 with App Router
- **Backend**: Convex BaaS (Backend as a Service)
- **Styling**: Tailwind CSS v4
- **Authentication**: LinkedIn OAuth integration
- **Communications**: SMS notifications, phone call integration
- **Real-time Features**: Convex WebSockets for chat functionality

## Target Users

**Primary Users**: Military veterans seeking mentorship or willing to provide mentorship
**User Personas**:
- **Mentees**: Veterans transitioning to civilian careers, seeking guidance on job searches, career changes, or personal development
- **Mentors**: Experienced veterans willing to share knowledge and provide guidance to their peers

## Core Features

### 1. Authentication & Onboarding
- LinkedIn OAuth integration for sign-up and login
- User profile creation during onboarding
- Account verification for veteran status

### 2. User Profiles
- **Profile Information**:
  - Personal details (name, location, military branch/rank)
  - Professional background (current role, industry, skills)
  - LinkedIn profile integration
  - Mentor availability toggle (available/unavailable for mentorship)
  - Profile photo and bio

- **Availability Calendar**:
  - Weekly availability settings for each day
  - Configurable time slots in 1-hour increments
  - Timezone support
  - Recurring availability patterns

### 3. Mentorship Request System
- **Request Creation**:
  - 1-hour time slot booking system
  - Date and time selection based on mentor availability
  - Request message/description field
  - Request reason/topic selection

- **Request Management**:
  - Accept/decline functionality for mentors
  - Request status tracking (pending, accepted, declined, completed)
  - Automatic SMS notifications for accepted requests
  - Email confirmations and reminders

### 4. Communication System
- **Phone Integration**:
  - Automated phone calls at scheduled mentorship times
  - Conference call system connecting mentor and mentee
  - Call duration tracking

- **Chat Feature** (Phase 2):
  - Real-time messaging using Convex WebSockets
  - Direct messages between mentor and mentee
  - Chat history and message persistence

### 5. Search & Discovery
- **Mentor Search**:
  - Search filters: industry, job title, location, military branch
  - Advanced filtering options: experience level, availability
  - Search results with mentor cards showing key information

- **Mentor Profile Modal**:
  - Overlay modal instead of separate pages
  - Mentor details, availability, and reviews
  - "Request Mentorship" button with availability selection

### 6. Dashboard
- **Personal Statistics**:
  - Mentorship calls completed (as mentor and mentee)
  - Requests sent and received
  - Request acceptance rate
  - Total mentorship hours provided/received
  - Upcoming scheduled mentorship sessions

- **Quick Actions**:
  - Manage availability
  - View pending requests
  - Access recent conversations

### 7. Review System
- **Post-Session Reviews**:
  - Email follow-up to mentees after sessions
  - In-platform review prompts
  - 5-star rating system with written feedback
  - Review display on mentor profiles

- **Review Management**:
  - Review moderation system
  - Response capability for mentors
  - Review aggregation and statistics

## User Flows

### 1. New User Onboarding
1. User clicks "Sign up with LinkedIn"
2. LinkedIn OAuth authentication
3. Profile setup wizard (personal info, military background, professional details)
4. Set mentorship preferences (mentor/mentee/both)
5. If mentor: configure availability calendar
6. Email verification and welcome message

### 2. Mentor Discovery & Request
1. User navigates to search page
2. Applies filters (industry, location, etc.)
3. Browses mentor results
4. Clicks on mentor card to open profile modal
5. Views availability calendar
6. Selects date/time and sends request with message
7. Receives confirmation and waits for response

### 3. Mentorship Session
1. Both parties receive SMS confirmation when request is accepted
2. Email reminders sent 24 hours and 1 hour before session
3. Automated phone calls initiated at scheduled time
4. Session conducted over phone
5. Session automatically ends after 1 hour
6. Follow-up email sent to mentee with review request

## Technical Requirements

### Authentication
- LinkedIn OAuth 2.0 integration
- JWT token management
- Session persistence
- Role-based access control

### Database Schema (Convex)
- **Users**: Profile information, authentication data
- **Availability**: Weekly schedules, time slots
- **Requests**: Mentorship requests with status tracking
- **Sessions**: Completed mentorship sessions
- **Reviews**: Session feedback and ratings
- **Messages**: Chat functionality (Phase 2)

### External Integrations
- **LinkedIn API**: Profile data import
- **SMS Service**: Twilio or similar for notifications
- **Phone Service**: Twilio Voice for automated calling
- **Email Service**: Transactional emails for notifications and reviews

### Performance Requirements
- Page load times under 3 seconds
- Real-time updates for availability and requests
- Responsive design for mobile and desktop
- 99.9% uptime for critical mentorship scheduling

## Security & Privacy
- HTTPS enforcement
- Data encryption at rest and in transit
- User data privacy controls
- Secure handling of veteran verification data
- GDPR compliance for data management

## Success Metrics
- **User Engagement**:
  - Monthly Active Users (MAU)
  - Session completion rate
  - Request acceptance rate
  - User retention rate

- **Platform Health**:
  - Average response time to mentorship requests
  - User satisfaction scores from reviews
  - Platform usage patterns and peak times
